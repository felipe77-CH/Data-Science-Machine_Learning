Importacion de los datos¶

La importacion de los datos parten del archivo "CSV". Un archivo CSV (Comma-Separated Values) es un formato de archivo que se utiliza para almacenar datos tabulares, como una hoja de clculo o una base de datos. En un archivo CSV, los datos se organizan en filas y columnas, y cada celda de la tabla est separada por un delimitador, comnmente una coma (,). Cada fila del archivo CSV representa una entrada de datos, y las columnas dentro de esa fila contienen los valores asociados con diferentes atributos o caractersticas. La primera fila a menudo se utiliza para los encabezados, que describen el contenido de cada columna.

Consideremos el siguente ejemplo:

Creacion de atributos¶

La "creacin de atributos" se refiere al proceso de generar nuevas caractersticas (o atributos) a partir de los datos existentes en un conjunto de datos. Estos nuevos atributos se calculan o derivan de las caractersticas originales y se pueden disear para proporcionar informacin adicional o para facilitar la tarea de aprendizaje automtico.

Ingeniera de Caractersticas: Esto implica crear nuevas caractersticas a partir de las existentes utilizando conocimiento del dominio o intuicin. Por ejemplo, si tienes la fecha de nacimiento de las personas, podras crear una nueva caracterstica para representar su edad.

Transformaciones Matemticas: Aplicar funciones matemticas a las caractersticas existentes para obtener nuevas. Por ejemplo, calcular el logaritmo o la raz cuadrada de una caracterstica.

Normalizacin o Estandarizacin: Ajustar las escalas de las caractersticas para que tengan una media de cero y una desviacin estndar de uno. Esto a menudo se realiza para asegurar que todas las caractersticas contribuyan de manera equitativa al modelo.

Binning o Discretizacin: Agrupar valores continuos en intervalos discretos. Esto puede facilitar la interpretacin y puede ser til en algoritmos que trabajan mejor con datos categricos.

Codificacin One-Hot: Convertir variables categricas en vectores binarios (0 o 1) para que puedan ser proporcionadas a los algoritmos de aprendizaje automtico que esperan datos numricos.

Interacciones entre Caractersticas: Crear nuevas caractersticas basadas en la interaccin de dos o ms caractersticas. Por ejemplo, si tienes las caractersticas "ancho" y "alto", podras crear una nueva caracterstica llamada "rea" multiplicando estas dos.

Creacion de columnas "Dummys"

La funcin "get_dummies" de pandas para realizar la codificacin one-hot de un DataFrame llamado df_nuevo. La codificacin one-hot se utiliza para convertir variables categricas en variables binarias (0 o 1) que pueden ser procesadas por algoritmos de aprendizaje automtico.

Cuando aplicas get_dummies(df_nuevo), pandas identificar automticamente las columnas categricas en el DataFrame df_nuevo y crear nuevas columnas binarias para cada categora en esas columnas categricas. Cada nueva columna binaria representa una categora diferente y tendr un valor de 1 si esa categora est presente en la fila correspondiente y 0 en caso contrario.

Por ejemplo, si df_nuevo contiene una columna llamada "Color" con valores como "Rojo", "Verde" y "Azul", get_dummies crear tres nuevas columnas llamadas "Color_Rojo", "Color_Verde" y "Color_Azul", y asignar 1 a la columna correspondiente para indicar el color presente en cada fila.

El resultado, df_nuevo, contendr las nuevas columnas binarias en lugar de las columnas originales que eran categricas. Esta transformacin es til para preparar datos categricos para su uso en modelos de aprendizaje automtico que requieren datos numricos en lugar de categricos.

Pre-Acondicionamiento: Reemplazar los valores faltantes en las columnas numricas especificadas

Este cdigo utiliza la tcnica de imputacin iterativa para reemplazar los valores faltantes en las columnas numricas especificadas en num_cols en los conjuntos de entrenamiento y prueba (X_train y X_test). Los resultados se imprimen al final, pero estn comentados. Puedes descomentar esas lneas segn sea necesario para inspeccionar los conjuntos de entrenamiento y prueba despus de la imputacin.

#Definir el numero de columnas donde vamos a realizar las operaciones
num_cols = ["pclass","age","sibsp","parch","fare","sex_female",]
imputer = impute.IterativeImputer()
#Colocamos a los datos de entrnamiento
imputed = imputer.fit_transform(X_train[num_cols])
X_train.loc[:, num_cols] = imputed
#Colocamos a los datos de prueba
imputed = imputer.transform(X_test[num_cols])
X_test.loc[:, num_cols] = imputed

NUM_COLS es una lista que contiene los nombres de las columnas que deseas imputar. Estas columnas contienen variables numricas en tus datos.

IMPUTER es un objeto creado a partir de la clase "IterativeImputer". Este objeto se utilizar para realizar la imputacin de valores faltantes en las columnas especificadas en num_cols.

imputed es una variable donde se almacenan los valores imputados. Se crea utilizando la funcin fit_transform del objeto imputer. Esta funcin toma como entrada las columnas numricas de X_train especificadas en num_cols y calcula los valores imputados. Luego, estos valores imputados se asignan a la variable imputed.

A continuacin, se actualiza el DataFrame X_train con los valores imputados. Se utiliza la funcin loc para seleccionar las mismas columnas numricas especificadas en num_cols, y se les asigna los valores imputados que se calcularon previamente en el paso anterior.

Luego, se aplica la funcin transform del objeto imputer a las mismas columnas numricas en el conjunto de prueba (X_test). Esto utiliza la informacin aprendida durante el ajuste en el conjunto de entrenamiento para imputar los valores faltantes en el conjunto de prueba y actualiza X_test con los valores imputados.

Finalmente, se repite el proceso de actualizacin del DataFrame X_test con los valores imputados utilizando la funcin loc.

Datos normalizados

Normalizar datos es un proceso importante en el preprocesamiento de datos para muchos algoritmos de aprendizaje automtico. Aqu hay algunas razones clave para normalizar los datos:

  • Escalas Consistentes: Al normalizar, puedes asegurarte de que todas las variables tengan la misma escala. Esto es crucial para algoritmos que son sensibles a las diferencias de escala entre las caractersticas, como las mquinas de soporte vectorial (SVM) o los mtodos basados en la distancia, como el k-vecino ms cercano (K-NN).

  • Mejora la Convergencia: Algoritmos basados en mtodos iterativos, como el descenso de gradiente en algoritmos de optimizacin, pueden converger ms rpido en datos normalizados. La normalizacin evita que las variables con valores grandes dominen la contribucin a las actualizaciones de parmetros.

  • Evita Problemas Numricos: Al normalizar, reduces el riesgo de problemas numricos, como la prdida de precisin debido a la magnitud de los nmeros.

  • Interpretacin ms Sencilla: La normalizacin facilita la interpretacin de los coeficientes o importancias en algunos modelos. Por ejemplo, en la regresin lineal, los coeficientes representan la contribucin promedio de una variable, y la normalizacin facilita la comparacin de estas contribuciones.

  • Mejora el Rendimiento: Algoritmos que utilizan medidas de distancia, como k-NN o k-medias, pueden funcionar mejor con datos normalizados, ya que las distancias se calculan de manera ms significativa.

Es importante sealar que no todos los algoritmos requieren normalizacin, y la necesidad de normalizacin puede depender del contexto especfico del problema y del algoritmo que se est utilizando. Sin embargo, en muchos casos, normalizar los datos es una buena prctica para mejorar la estabilidad y el rendimiento de los modelos.

Estimadores de prueba : CLASIFICADOS DUMMY

Un clasificador "Dummy (Dummy Classifier)" es un tipo de clasificador que se utiliza como referencia bsica o lnea base para evaluar el rendimiento de otros clasificadores ms complejos. A diferencia de los clasificadores ms avanzados, un clasificador Dummy toma decisiones de clasificacin de manera simple y siguiendo reglas predefinidas, sin aprender de los datos.

Existen dos tipos principales de clasificadores Dummy:

Clasificador Dummy Estratificado (Stratified): Este tipo de clasificador asigna las clases a las instancias en proporciones proporcionales a la frecuencia de las clases en el conjunto de entrenamiento. Por ejemplo, si tienes un conjunto de datos con dos clases, una representando el 70% de las instancias y la otra el 30%, un clasificador Dummy Estratificado asignar las clases de manera que refleje esa proporcin.

Clasificador Dummy Aleatorio (Random): Este clasificador toma decisiones de clasificacin de manera completamente aleatoria. No tiene en cuenta la distribucin de clases en el conjunto de entrenamiento y, por lo tanto, sus predicciones son completamente al azar. Este tipo de clasificador se utiliza para establecer un punto de referencia muy bsico.

Cuando se realiza un aprendizaje supervisado, una simple comprobacin de cordura consiste en comparar el estimador propio con reglas simples. DummyClassifier implementa varias de estas estrategias simples para la clasificacin:

  • stratified: genera predicciones aleatorias respetando la distribucin de la clase del conjunto de entrenamiento.

  • most_frequent: siempre predice la etiqueta ms frecuente en el conjunto de entrenamiento.

  • prior: siempre predice la clase que maximiza la clase anterior (como most_frequent) y predict_proba devuelve la clase anterior.

  • uniform: genera predicciones uniformemente al azar.

  • constant: siempre predice un valor constante proporcionado por el usuario.

Una motivacin importante de este mtodo es la puntuacin F1, cuando la clase positiva est en la minora.
